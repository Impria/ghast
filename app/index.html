Ghast

A terrifyingly simple CSS framework

Download

So, is Ghast for you?

Ghast was designed to provide everything you need to get started with your own website in a small, responsive package. It was built to be easy to understand, maintain and adapt to your needs.


If you enjoy Ghast or think it's a cool project, help spread the word out by sharing it with your fam!

Getting Started

There are some ways to get started. First, see all download options available below, then choose the most suitable option for your need. Now you should add the main file of the Ghast and the CSS Reset in the header of your project. Just that!

Install with npm

Ghast is available to install using npm.

$ npm install ghast

Install with Bower

Ghast is also available to install using Bower.

$ bower install ghast

What's included

Once downloaded, extract the compressed folder to see the main file in the uncompressed and minified version.

Typography

CSS3 introduces a few new units, including the rem unit, which stands for "root em". The rem unit is relative to the font-size of the root element <html>. That means that we can define a single font size on the root element, and define all rem units to be a percentage of that. The typography has font-size defined as 1.6rem (16px) and line-height as 1.6 (24px). Ghast uses the font-family Roboto, created by Christian Robertson, and provided by Google.

Grids

The Grid is a fluid system with a max width of 70.0rem (1120px), that shrinks with the browser/device at smaller sizes. The max width can be changed with one line of CSS and all cols will resize accordingly. Ghast is different than most because of its use of the CSS Flexible Box Layout Module standard. The advantage is the simplicity, and we know that a functional code is very important for maintainability and scalability. Simply add cols you want in a row, and they'll evenly take up the available space. If you want three cols, add three cols, if you want five cols, add five cols. There is no restriction on number of cols, but we advise you to follow a design pattern of grid system. See more tips on best practices in the tips.

Mobile First

The Mobile First is the design strategy that takes priority development for mobile devices like smartphones and tablets. It means all styles outside of a media queries apply to all devices, then larger screens are targeted for enhancement. This prevents small devices from having to parse tons of unused CSS. Ghast use some breakpoints like these:

  • Larger than Mobile screen: 40.0rem (640px)
  • Larger than Tablet screen: 80.0rem (1280px)
  • Larger than Desktop screen: 120.0rem (1920px)
/* Mobile First Media Queries */

/* Base style */
{ ... }

/* Larger than mobile screen */
@media (min-width: 40.0rem) { ... }

/* Larger than tablet screen */
@media (min-width: 80.0rem) { ... }

/* Larger than desktop screen */
@media (min-width: 120.0rem) { ... }

----

/* Sass media queries  */

/* Larger than mobile screen */
+media(mobile)

/* Larger than tablet screen */
+media(tablet)

/* Larger than desktop screen */
+media(desktop)

Extending The Inheritances

The style of an element can be defined in several different locations, which interact in a complex way. It is form of interaction makes CSS powerful, but it can make it confusing and difficult to debug.

Default .button Outline .button Clear .button
.primary .primary .primary
.button-small .button-small .button-small
.button-large .button-large .button-large 
/* Extending The Inheritances */

/* Applying color variation */
.button-blue {
  background-color: #536DFE;
  border-color: #536DFE;
}
.button-blue.outline {
  color: #536DFE;
}
.button-blue.clear {
  color: #536DFE;
}

/* Applying size variation */
.button-small {
  font-size: .8rem;
  height: 2.8rem;
  line-height: 2.8rem;
  padding: 0 1.5rem;
}

.button-large {
  font-size: 1.4rem;
  height: 4.5rem;
  line-height: 4.5rem;
  padding: 0 2rem;
}
